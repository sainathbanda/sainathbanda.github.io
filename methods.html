Ordinary Least Squares (OLS) Regression
Beginner–Intermediate
Econometrics
▾
The foundation of econometrics. Estimates the linear relationship between a continuous dependent variable and one or more independent variables. Starting point for almost all empirical research in agricultural economics.
Use When
Dependent variable is continuous
Data Type
Cross-section, Panel
Software
STATA, R, Python, Excel
Output
Coefficients, R², p-values

When to Use

Use OLS when your dependent variable is continuous (e.g. farm income, crop yield, milk production, water use efficiency) and you want to estimate the effect of one or more independent variables. It is the workhorse of empirical economics.

Common Applications in Agri-Economics

  • Determinants of farm income / crop yield
  • Effect of irrigation on productivity
  • Impact of technology adoption on output
  • Price analysis and cost of production studies

Key Limitations

OLS breaks down when: dependent variable is binary or censored; errors are heteroskedastic; there is multicollinearity; or endogeneity exists. Always run diagnostic tests before reporting results.

✓
Linearity — relationship between X and Y is linear. Test: residual vs fitted plot.
✓
No perfect multicollinearity — independent variables are not perfectly correlated. Test: VIF < 10.
✓
Homoskedasticity — constant variance of errors. Test: Breusch-Pagan test. Fix: robust standard errors.
✓
No autocorrelation — errors are uncorrelated. Test: Durbin-Watson (time series). Fix: HAC standard errors.
⚠
No endogeneity — regressors uncorrelated with error term. If violated, use IV/2SLS instead.
⚠
Normal errors — needed for inference in small samples. Large samples: CLT applies.
STATA
* Basic OLS regression
reg farm_income land_size irrigation education age, robust

* View results
outreg2 using results.doc, replace word

* Check multicollinearity
vif

* Breusch-Pagan test for heteroskedasticity
estat hettest

* Ramsey RESET test for functional form
estat ovtest

* Standardised coefficients (beta coefficients)
reg farm_income land_size irrigation education, beta

* OLS with clustered standard errors
reg farm_income land_size irrigation, vce(cluster district_id)

Python
import pandas as pd
import statsmodels.formula.api as smf
from statsmodels.stats.outliers_influence import variance_inflation_factor

# Load data
df = pd.read_csv("farm_data.csv")

# OLS with robust standard errors
model = smf.ols('farm_income ~ land_size + irrigation + education + age', 
               data=df).fit(cov_type='HC3')
print(model.summary())

# VIF for multicollinearity check
X = df[['land_size', 'irrigation', 'education']]
vif = pd.DataFrame()
vif["feature"] = X.columns
vif["VIF"] = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]
print(vif)

Logit & Probit Models
Intermediate
Econometrics
▾
Used when the dependent variable is binary (0/1). Estimates probability of adoption, participation, or occurrence. Logit uses logistic distribution; Probit uses normal distribution. Most common in technology adoption studies.
Use When
Dependent variable is binary (0/1)
Output
Odds ratios / Marginal effects
Key Statistic
McFadden's R², LR test
Report
Marginal effects (not coefficients)

When to Use

Use when your dependent variable takes only two values: adoption (yes/no), participation (yes/no), credit access (yes/no), organic farming (yes/no). Never use OLS for binary outcomes — it gives predicted values outside [0,1].

Logit vs Probit

Logit: easier to interpret (odds ratios), slightly thicker tails. Probit: theoretically preferred when errors are normally distributed. In practice, results are nearly identical — use Logit for most agricultural economics work.

Critical: Always Report Marginal Effects

Raw coefficients from Logit/Probit are not interpretable directly. Always compute and report Average Marginal Effects (AME) which show the change in probability for a unit change in X.

✓
Binary outcome — DV must be 0 or 1 (not ordered, not count).
✓
No perfect multicollinearity — test with VIF on auxiliary OLS.
✓
Large sample — ML estimation needs n > 100. Rule: 10 observations per predictor.
⚠
No separation — if a predictor perfectly predicts the outcome, estimation fails. Check with tab X outcome in STATA.
⚠
No endogeneity — if present, use IV Probit or Bivariate Probit.
STATA
* Logit model
logit adoption land_size credit_access extension_contact age education, robust

* Average Marginal Effects (AME) — always report these
margins, dydx(*)

* Probit model
probit adoption land_size credit_access extension_contact age, robust
margins, dydx(*)

* Goodness of fit
estat gof

* Classification table (sensitivity/specificity)
estat classification

* Ordered Logit (for ordinal DV: low/medium/high)
ologit satisfaction income land_size education
margins, dydx(*) predict(outcome(1))

Tobit Model
Intermediate
Econometrics
▾
Used when the dependent variable is censored — many observations are piled at zero or an upper limit. Classic example: expenditure on fertiliser (many farmers spend zero). OLS underestimates effects in censored data.
Use When
DV censored at 0 or upper limit
Common Issue
Many zeros in DV
Alternative
Double-hurdle model (Cragg)
Key Output
Marginal effects for E(y|y>0)

When to Use

Use Tobit when your dependent variable has a pile-up of observations at zero (left-censored) and positive values beyond that. Examples in agricultural economics: fertiliser expenditure, pesticide use, credit amount borrowed, remittances sent.

Tobit vs Double-Hurdle

Tobit: assumes the same process drives both the decision to participate AND the level. Double-Hurdle (Cragg): separates the participation decision (Probit) from the level decision (truncated regression). Double-hurdle is more realistic for most agricultural data.

STATA
* Standard Tobit (left-censored at 0)
tobit fertiliser_exp land_size income education, ll(0)

* Marginal effects
margins, dydx(*)

* Right-censored Tobit
tobit market_share price quality, ul(100)

* Double-hurdle model (requires -craggit- package)
* ssc install craggit
craggit fertiliser_exp land_size income education credit_access

* Two-part model alternative
probit fertiliser_use land_size income education  /* participation */
reg fertiliser_exp land_size income education if fertiliser_use==1  /* level */

Panel Data — Fixed & Random Effects
Intermediate
Econometrics
▾
Panel data has observations for the same units over multiple time periods. Fixed Effects (FE) controls for unobserved unit-level heterogeneity. Random Effects (RE) is more efficient if heterogeneity is uncorrelated with regressors.
Data Required
Same units × multiple time periods
FE Advantage
Controls time-invariant confounders
Key Test
Hausman test (FE vs RE)
Best Package
plm (R), xtreg (STATA)

FE vs RE — Which to Choose?

Fixed Effects: Use when you suspect the unit-level unobservables (farmer ability, soil quality) are correlated with your regressors. FE removes all time-invariant variation — you cannot estimate effects of time-invariant variables (gender, caste).

Random Effects: Use when unobservables are uncorrelated with regressors. More efficient. Can estimate time-invariant variable effects.

Hausman Test: Always run this. If significant (p < 0.05) → use FE. If not significant → RE is acceptable.

STATA
* Declare panel data
xtset farmer_id year

* Fixed Effects
xtreg yield rainfall fertiliser price, fe robust

* Random Effects
xtreg yield rainfall fertiliser price, re robust

* Hausman test (store FE and RE first)
xtreg yield rainfall fertiliser, fe
estimates store fe_model
xtreg yield rainfall fertiliser, re
estimates store re_model
hausman fe_model re_model

* Two-way FE (unit + time fixed effects)
xtreg yield rainfall fertiliser i.year, fe robust

* First difference estimator
xtreg D.yield D.rainfall D.fertiliser, fe
R
library(plm); library(lmtest)

# Declare panel data
pdata <- pdata.frame(mydata, index = c("farmer_id", "year"))

# Fixed Effects
fe_model <- plm(yield ~ rainfall + fertiliser + price,
                data = pdata, model = "within")
coeftest(fe_model, vcov = vcovHC(fe_model, type = "HC3"))

# Random Effects
re_model <- plm(yield ~ rainfall + fertiliser + price,
                data = pdata, model = "random")

# Hausman test
phtest(fe_model, re_model)

# Two-way FE
twfe <- plm(yield ~ rainfall + fertiliser,
            data = pdata, model = "within", effect = "twoways")

Difference-in-Differences (DiD)
Intermediate–Advanced
Impact Evaluation
▾
Estimates causal effect of a policy/intervention by comparing the change in outcomes before and after treatment between a treatment group and a control group. Gold standard for policy evaluation when RCT is not feasible.
Identifies
Causal ATT
Requires
Pre & post data, treatment & control
Key Assumption
Parallel trends
Modern Extension
Callaway-Sant'Anna (staggered DiD)

The DiD Logic

DiD = (Post_treatment − Pre_treatment) − (Post_control − Pre_control). By subtracting the control group's trend, we remove the confounding effect of time. The key identifying assumption is that both groups would have followed parallel trends in absence of treatment.

Applications in Agri-Economics

  • Impact of PM-KISAN cash transfers on farm income
  • Effect of minimum support price change on crop area
  • Impact of irrigation scheme on productivity
  • Effect of FPO formation on farmer income

Modern DiD (Post-2020)

With staggered treatment adoption (different units treated at different times), the classical 2x2 DiD estimator is biased. Use Callaway-Sant'Anna (2021) or Sun-Abraham (2021) estimators instead.

✓
Parallel Trends — most critical. Pre-treatment trends must be parallel between groups. Test: event study plot; visual inspection of pre-treatment periods.
✓
No anticipation — treatment group should not change behaviour before treatment occurs.
✓
SUTVA — no spillovers between treatment and control groups.
⚠
Compositional stability — the composition of treatment/control groups should not change over time.
STATA
* Basic DiD — interaction term approach
gen did = treated * post
reg outcome treated post did controls, robust
* Coefficient on -did- is the ATT

* DiD with fixed effects (preferred)
xtset unit_id year
xtreg outcome treated##post controls i.year, fe robust

* Event study plot
gen rel_time = year - treatment_year
xtreg outcome i.rel_time controls, fe robust
coefplot, vertical yline(0) xline(-0.5) /* Visual parallel trends test */

* Staggered DiD — Callaway-Sant'Anna
* ssc install csdid
csdid outcome controls, ivar(unit_id) time(year) gvar(first_treated)
csdid_plot
R
library(fixest)   # for feols (fast FE)
library(did)      # Callaway-Sant'Anna

# Basic DiD with TWFE
did_model <- feols(outcome ~ treated:post | unit_id + year,
                   data = mydata, vcov = "cluster")
summary(did_model)

# Event study (iplot from fixest)
es <- feols(outcome ~ i(rel_time, treated, ref = -1) | unit_id + year,
            data = mydata)
iplot(es, main = "Event Study — Parallel Trends Test")

# Callaway-Sant'Anna (staggered DiD)
cs <- att_gt(yname = "outcome", tname = "year", 
            idname = "unit_id", gname = "first_treated",
            data = mydata, control_group = "nevertreated")
summary(aggte(cs, type = "dynamic"))
ggdid(cs)

Propensity Score Matching (PSM)
Intermediate–Advanced
Impact Evaluation
▾
Controls for selection bias by matching treated units to similar untreated units based on observable characteristics. Estimates Average Treatment Effect on the Treated (ATT). Widely used in technology adoption and programme evaluation.
Identifies
ATT (Average Treatment Effect on Treated)
Key Assumption
Conditional independence (CIA)
Matching Methods
Nearest neighbour, Kernel, Radius
Sensitivity Test
Rosenbaum bounds

Steps in PSM

  • Step 1: Estimate propensity score using Logit/Probit on all relevant covariates
  • Step 2: Match treated to control units using nearest neighbour / kernel matching
  • Step 3: Check covariate balance after matching (standardised bias < 10%)
  • Step 4: Estimate ATT on matched sample
  • Step 5: Rosenbaum bounds sensitivity analysis

PSM vs DiD

PSM requires only cross-sectional data but relies on the strong CIA assumption (no unobservables). DiD requires panel data but controls for time-invariant unobservables. PSM+DiD combined is the most robust approach when panel data is available.

STATA
* Install psmatch2
* ssc install psmatch2

* Step 1: Estimate propensity score
logit treated land_size age education credit_access market_distance

* Step 2: Nearest-neighbour matching (1:1, with replacement)
psmatch2 treated land_size age education credit_access, ///
         outcome(farm_income) n(1) common

* Step 3: Check balance
pstest land_size age education credit_access, both

* Kernel matching
psmatch2 treated land_size age education, ///
         outcome(farm_income) kernel bwidth(0.06)

* Rosenbaum bounds sensitivity test
* ssc install rbounds
rbounds farm_income, gamma(1(.1)2)
R
library(MatchIt); library(cobalt)

                     data = mydata, method = "nearest", ratio = 1,
                     distance = "logit")
summary(match_out)  # check balance

# Plot balance
love.plot(match_out, threshold = 0.1)

# Get matched data

# Estimate ATT on matched sample
att_model <- lm(farm_income ~ treated + land_size + age + education,
                data = matched_data, weights = weights)
coeftest(att_model, vcov = vcovHC(att_model))

# Kernel matching
match_kern <- matchit(treated ~ land_size + age + education,
                      data = mydata, method = "full", distance = "logit")

Discrete Choice Experiment (DCE)
Advanced
Survey Methods
▾
Stated preference method that asks respondents to choose between alternatives described by attributes. Used to estimate WTP, preference weights, and design optimal programmes. Highly relevant for carbon credit market design and ecosystem services.
Theoretical Base
Random Utility Theory
Estimation
Conditional Logit, MXL
Key Output
WTP, preference weights
Design Software
Ngene, R (idefix)

DCE Steps

  • 1. Attribute identification — qualitative research (FGDs) to identify relevant attributes and levels
  • 2. Experimental design — D-efficient design using Ngene or idefix in R
  • 3. Survey instrument — choice sets with opt-out option; pilot test
  • 4. Data collection — face-to-face interviews recommended for rural India
  • 5. Estimation — Conditional Logit (homogeneous) → Mixed Logit (heterogeneity) → Latent Class

Applications in Agri-Economics

  • Farmer preferences for carbon credit programme design
  • WTP for improved seed varieties
  • Consumer preferences for organic produce
  • Farmer preferences for crop insurance attributes

Experimental Design Guidelines

Number of attributes: 4–6 maximum (cognitive burden increases rapidly beyond this)

Number of levels per attribute: 2–4 levels. More levels = more information but more complex survey.

Number of choice sets: 8–16 per respondent typically. Use Ngene to find D-efficient design.

Payment vehicle: Be specific — "per season per acre" is better than vague payment framing.

NGENE Script — D-efficient DCE Design
; Carbon Credit Programme DCE Design
; Attributes: Payment, Duration, Monitoring, Aggregator
Design
alts = alt1, alt2, opt_out
rows = 16
eff = (mnl, d, mean)
model:
U(alt1) = b_pay[0.1]*payment[500,1000,2000] +
          b_dur[0.2]*duration[1,3,5] +
          b_mon[-0.1]*monitoring[1,2] +
          b_agg[0.15]*aggregator[0,1];
U(alt2) = b_pay*payment + b_dur*duration +
          b_mon*monitoring + b_agg*aggregator;
U(opt_out) = asc_opt[0];

ARIMA / SARIMA — Time Series Forecasting
Intermediate–Advanced
Time Series
▾
ARIMA (AutoRegressive Integrated Moving Average) models univariate time series for forecasting. SARIMA adds seasonal components. Used for agricultural price forecasting, production trends, and rainfall analysis.
Parameters
ARIMA(p,d,q)
p
AR order (PACF)
d
Differencing order
q
MA order (ACF)

Step-by-Step

  • 1. Stationarity test: ADF / KPSS test. If non-stationary → difference the series (d=1 usually)
  • 2. Identify p and q: Plot ACF (→q) and PACF (→p) of stationary series
  • 3. Fit model: Use auto.arima() in R for automatic selection
  • 4. Diagnostics: Ljung-Box test on residuals (should be white noise)
  • 5. Forecast: Out-of-sample forecast with confidence intervals
STATA
* Declare time series
tsset date

* ADF unit root test
dfuller price, trend lags(4)
kpss price   /* KPSS test */

* ACF and PACF plots
ac price, lags(24)
pac price, lags(24)

* Fit ARIMA(1,1,1)
arima price, arima(1,1,1)

* Seasonal ARIMA
arima price, sarima(1,1,1)(1,1,1,12)

* Forecast 12 periods
predict price_hat
tsappend, add(12)
predict forecast, dynamic(.)

Analytical Hierarchy Process (AHP)
Intermediate
Survey Methods
▾
Multi-criteria decision analysis method that prioritises alternatives through pairwise comparisons. Used in your own published research for prioritising sustainable urban dairy management strategies. Excellent for policy prioritisation studies.
Scale
Saaty's 1–9 scale
Consistency
CR < 0.10 required
Output
Priority weights, rankings
Extension
Fuzzy-AHP for uncertainty

Steps

  • 1. Define goal, criteria, and alternatives hierarchically
  • 2. Collect pairwise comparison matrices from experts
  • 3. Calculate priority vectors (eigenvector method)
  • 4. Check Consistency Ratio (CR < 0.10)
  • 5. Aggregate across experts (geometric mean)
  • 6. Calculate overall priority rankings

Saaty's Comparison Scale

1 = Equal importance · 3 = Moderate superiority · 5 = Strong superiority · 7 = Very strong · 9 = Extreme superiority · 2,4,6,8 = Intermediate values

R
library(ahp)     # main AHP package
library(ahpsurvey) # for survey-based AHP

# Define pairwise comparison matrix (criteria)
# Saaty scale: 1=equal, 3=moderate, 5=strong, 7=very strong, 9=extreme
criteria_matrix <- matrix(c(
  1,   3,   5,   7,
  1/3, 1,   3,   5,
  1/5, 1/3, 1,   3,
  1/7, 1/5, 1/3, 1
), nrow = 4, byrow = TRUE)

rownames(criteria_matrix) <- colnames(criteria_matrix) <- 
  c("Economic", "Environmental", "Social", "Technical")

# Calculate priority weights using eigenvector method
eigenvalues <- eigen(criteria_matrix)
weights <- Re(eigenvalues$vectors[,1]) / sum(Re(eigenvalues$vectors[,1]))
names(weights) <- rownames(criteria_matrix)
print(round(weights, 4))

# Consistency Ratio
lambda_max <- Re(eigenvalues$values[1])
n <- nrow(criteria_matrix)
CI <- (lambda_max - n) / (n - 1)
RI <- c(0,0,0.58,0.90,1.12,1.24,1.32,1.41,1.45,1.49)[n]
CR <- CI / RI
cat("Consistency Ratio:", round(CR,4), if(CR<0.1) "✓ Acceptable" else "✗ Revise matrix")
Python
import numpy as np

def ahp_weights(matrix):
    # Normalize columns then average rows
    col_sums = matrix.sum(axis=0)
    norm = matrix / col_sums
    weights = norm.mean(axis=1)
    
    # Consistency check
    n = matrix.shape[0]
    weighted_sum = np.dot(matrix, weights)
    lambda_max = np.mean(weighted_sum / weights)
    CI = (lambda_max - n) / (n - 1)
    RI = [0,0,0.58,0.90,1.12,1.24,1.32,1.41][n-1]
    CR = CI / RI
    return weights, CR

# Pairwise comparison matrix
matrix = np.array([
    [1,   3,   5,   7],
    [1/3, 1,   3,   5],
    [1/5, 1/3, 1,   3],
    [1/7, 1/5, 1/3, 1]
])

weights, CR = ahp_weights(matrix)
print(f"Priority weights: {np.round(weights,4)}")
print(f"Consistency Ratio: {CR:.4f} {'✓' if CR<0.1 else '✗ Revise'}")

Computable General Equilibrium (CGE) Modelling
Expert
Advanced Modelling
▾
Economy-wide simulation model representing all markets simultaneously. Used for analysing trade policy shocks, tariff changes, and distributional consequences across sectors and income groups. Requires GAMS or GEMPACK software.
Software
GAMS, GEMPACK
Data Needed
Social Accounting Matrix (SAM)
Standard Model
GTAP, PEP-1-1
Training
IFPRI + ICAR-NAARM

Components of a CGE Model

  • Social Accounting Matrix (SAM): Data backbone — captures all transactions between sectors, households, government, and rest of world
  • Production functions: CES/Leontief — how sectors combine labour, capital, land, and intermediates
  • Demand system: LES or CDE — how households allocate income across goods
  • Closure rules: Determine which variables are endogenous vs exogenous
  • Elasticities: Trade (Armington), substitution (CES σ), income elasticities

Applications

  • Impact of US tariff shocks on Indian agriculture (your own published work!)
  • Effect of food subsidy reforms on rural households
  • Trade liberalisation and agricultural competitiveness
  • Carbon tax impact on agricultural sectors

Bibliometric Analysis
Intermediate
Data Science
▾
Quantitative analysis of scientific literature — co-authorship, keyword co-occurrence, citation networks. Used in your published blockchain paper (JADEE, IF 2.3). Growing rapidly as a research methodology in agricultural economics.
Data Sources
Web of Science, Scopus
Tools
VOSviewer, bibliometrix (R)
Analysis Types
Co-authorship, co-citation, keyword
Export Format
.txt (WoS), .csv (Scopus)

Types of Bibliometric Analysis

  • Performance analysis: Most cited papers, productive authors, leading journals, country output
  • Co-authorship network: Collaboration patterns between researchers and countries
  • Keyword co-occurrence: Research themes and intellectual structure of a field
  • Co-citation analysis: Intellectual foundations of a research area
  • Bibliographic coupling: Emerging research fronts
R — bibliometrix + biblioshiny
library(bibliometrix)

# Load Web of Science export (.txt format)
M <- convert2df(file = "wos_export.txt", dbsource = "wos", format = "plaintext")

# Bibliometric summary
results <- biblioAnalysis(M)
summary(results, k=10, pause=FALSE)

# Most cited papers
citations(M, field="article", sep=";")$Cited[1:10]

# Keyword co-occurrence network
NetMatrix <- biblioNetwork(M, analysis="co-occurrences", 
                           network="keywords", sep=";")
net <- networkPlot(NetMatrix, normalize="salton", n=50,
                   Title="Keyword Co-occurrence Network", 
                   type="fruchterman", labelsize=1.5)

# Thematic evolution
thematicEvolution(M, field="ID", years=c(2010,2015,2020,2024))

# Interactive Shiny dashboard
biblioshiny()  # Opens browser dashboard
